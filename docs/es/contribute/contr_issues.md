# Reportar issues en GitHub

Además de contribuir con código y documentación, otra buena forma de ayudar al paquete es reportar issues. Un issue puede tratarse de una solicitud de funcionalidad, un reporte de bug o cualquiera de tus sugerencias útiles.

Usamos GitHub para llevar el seguimiento de los issues. Puedes ver todos los issues pasados y actuales (abiertos) simplemente visitando [esta página](https://github.com/pydap/pydap/issues).

Antes de reportar un nuevo issue, asegúrate de seguir estos dos pasos simples:

1. Lee las secciones relevantes de la documentación;
2. Usa la barra de búsqueda para ver si el issue ya fue reportado.

Si no puedes encontrar ningún issue o discusión previa relevante para tu problema, considera abrir uno nuevo. Al reportar un issue, también ayudas a otras personas que están en la misma situación. Si dedicas suficiente esfuerzo a trabajar en el problema y aun así no puedes resolverlo, debe haber algo confuso o incorrecto en el código o en la documentación. Agradeceremos mucho que lo señales.

Por último, recomendamos seguir prácticas comunes al describir el issue:

- sé conciso.
- sé amable. ¿Por qué importa esto en la comunidad de código abierto? Lee el [BlogPost](https://jacobtomlinson.dev/posts/2022/dont-be-that-open-source-user-dont-be-me/) de Jacom Tomlinson.
- incluye un `ejemplo mínimo reproducible` que recree el problema.
- describe el comportamiento esperado o el nuevo comportamiento deseado.
